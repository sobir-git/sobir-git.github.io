Binary Search once and for all

Thu 06 May 2021
By Sobir Bobiev
Category: data structures & algorithms

This is inspired by the following youtube tutorial.

Binary search is so intuitive but very tricky to code. I consider two main difficulties:

  1. Tricky to ensure termination, many times it falls into an infinite loop.
  2. Knowing exactly what to search.

The first issue is arithmetical, you have to make sure each time the search interval is strictly shortening.

The second issue has to do with the way you formulate your search objective. However, all binary search problems can be converted to the following problem:

Consider a binary array with the following property: no true comes before any false: e.g. FFFFFFTTTT, FFFFF, TTT

Then, the binary search problem is to find the leftmost insertion location for another true.

We end with the following python psuedocode:

def bisect(arr: list) -> int:
    '''Find the leftmost insertion location of 
    another True in the sorted binary array.'''

    lo = 0
    hi = len(arr)

    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] is True:
            hi = mid
        else:  # arr[mid] is False
            lo = mid + 1

    return lo

assert bisect([False, False, False, True]) == 3
assert bisect([False, False, False]) == 3
assert bisect([True, True, True]) == 0
